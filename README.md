# Knowledge Mining Agent
![](https://giga.chat/files/public/generated/f7dd4499-b4d3-4559-83cd-45ad9a02b559)

**Автономный ИИ-агент для управления базой знаний.**

Этот агент переводит управление корпоративной базой знаний из ручного режима в режим постоянного агентного сопровождения. Он автономно строит, совершенствует и актуализирует знания, используя подход **LLM Wiki**.

Разработан на базе фреймворка **Ouroboros** как единый агент-оркестратор, который управляет набором специализированных инструментов для работы с информацией.

## 1. 🚀 Быстрый старт за 4 шага

### Шаг 1: Клонируйте репозиторий
```bash
git clone https://github.com/netwise-team/knowledge_mining_agent.git
cd knowledge_mining_agent
```

### Шаг 2: Настройте и активируйте виртуальное окружение, а затем установите зависимости
```bash
python3.11 -m venv .venv      # Python >= 3.10
source .venv/bin/activate     # Для Windows: .venv\Scripts\activate

python -m pip install --upgrade pip setuptools wheel
python -m pip install -r requirements.txt
python -m pip install -e . --no-deps
```

### Шаг 3: Запустите агента и UI
```bash
ouroboros server
```
Затем откройте в браузере адрес: http://127.0.0.1:8765.

### Шаг 4: Пройдите предварительную настройку агента

* Провайдеры и модели
* Безопасность и бюджет
* Сеть и сервер
* Навыки, в т.ч. бот Telegram

## 2. 🧠 Возможности агента


| Функция | Описание |
| :--- | :--- |
| **Управление знаниями (LLM Wiki)** | Автономное построение, структурирование и совершенствование базы знаний. Проверка противоречий, пробелов и контроль актуальности данных. |
| **Поиск и ответы (SynthAdoc)** | Ответы на вопросы по базе знаний с генерацией саммари и дайджестов на основе синтетических документов и внутреннего контекста. |
| **Автоматический сбор публикаций** | Интеграция с внешними источниками (RSS, API, веб-поиск) для автоматического пополнения базы знаний актуальными данными из открытых источников. |
| **Автономное выполнение** | Работа по расписанию (cron) или в ответ на события (например, появление новой публикации). |
| **Оркестрация (Ouroboros)** | Единый агент-оркестратор управляет всеми инструментами, обрабатывает запросы через чат-интерфейс и выполняет фоновые задачи. |
| **Чат-интерфейс** | Простое и интуитивное общение с агентом для управления задачами, вопросов и получения статуса работы. |

## 3. ⚙️ Архитектура
Проект построен на основе трёх ключевых слоёв:

1. Ядро оркестрации (Ouroboros) : Обеспечивает самоизменение, постоянную идентичность агента, многоуровневую безопасность, поддержку множества LLM-провайдеров (OpenRouter, OpenAI, GigaChat, локальные модели) и работу через веб-интерфейс/CLI.

2. Инструменты управления знаниями:
- SynthAdoc: Модуль для генерации ответов и синтеза информации из базы знаний.
- LLM Wiki Engine: Набор промптов и логика для построения, проверки противоречий и обновления базы знаний в формате вики-статей.

3. Коннекторы к источникам данных:

Модули для сбора данных из RSS-лент, API сторонних сервисов, веб-скрейпинга и обработки загруженных документов.

```
[Чат-интерфейс / CLI]  →  [Агент-оркестратор (Ouroboros)]
│
▼
┌───────────────────────┼───────────────────────┐
▼                       ▼                       ▼
[Инструмент: Сбор данных]  [Инструмент: LLM Wiki]  [Инструмент: Поиск/Ответ]
(Источники, RSS, API)     (Построение знаний)     (SynthAdoc)
```

## 4. ⚙️ Конфигурация

Настройка агента производится через веб-интерфейс (Settings) или переменные окружения в файле .env.

Основные настройки:

```bash
OPENROUTER_API_KEY / OPENAI_API_KEY: Ключи для доступа к моделям.
KNOWLEDGE_BASE_PATH: Путь к директории, где хранится база знаний (по умолчанию ~/Ouroboros/data/knowledge/).
AUTO_UPDATE_SCHEDULE: Расписание в формате cron для автоматического обновления базы знаний.
SOURCES_CONFIG: Путь к YAML-файлу со списком источников данных для мониторинга.
GITHUB_TOKEN: Токен для резервного копирования и синхронизации базы знаний с удаленным репозиторием.
```
